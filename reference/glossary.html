Python Glossary

Essential Python terminology. Updated as new concepts are learned.

Core Concepts

Variable
A name that references an object. Created with =: name = "Alice". Variables are labels on objects, not containers.
Object Identity (id())
Every object has a unique identity (memory address). id(x) returns it. Never changes for the object's lifetime.
is
Identity comparison. a is b is True only when a and b reference the exact same object. Different from == (value comparison).
Mutable
An object whose value can change after creation. Examples: list, dict, set.
Immutable
An object whose value cannot change after creation. Examples: int, float, str, tuple, bool.
Assignment
Binds a name to an object: x = 5. The right side is evaluated first, then the name is bound to the result.
Augmented Assignment
Combines operation and assignment: x += 1. For mutable objects, may operate in-place. Equivalent to x = x + 1 but x is evaluated only once.
Tuple Unpacking
Assigning multiple variables from an iterable: a, b, c = [1, 2, 3] or swapping: a, b = b, a.
Star Unpacking
Using * to capture remaining items: first, *rest = [1, 2, 3] → first=1, rest=[2, 3].
Type Annotation
Syntax for hinting variable types: name: str = "Alice". Ignored at runtime but used by linters and type checkers.
del
Removes a name binding (not the object itself). The object persists if other names reference it.
snake_case
Naming convention for Python variables and functions: lowercase with underscores between words. my_variable_name.
Truthiness
Every object's boolean value. False: None, False, zero, empty sequences/collections. Everything else is True.
Numeric Types
int (unlimited precision), float (double-precision), complex (real + imag). All immutable.
Sequence Types
Ordered collections with index access. list (mutable), tuple (immutable), range (immutable, memory-efficient).
Set Types
Unordered collections of unique, hashable elements. set (mutable), frozenset (immutable, hashable).
Mapping Type
dict — key-value pairs, insertion-ordered (3.7+), keys must be hashable.
Binary Types
bytes (immutable), bytearray (mutable), memoryview (zero-copy buffer access).
isinstance()
Checks if an object is an instance of a type: isinstance(x, int). Preferred over type(x) is int because it handles subclasses correctly.
Hashable
An object with a hash value that never changes. Immutable types are hashable. Required for dict keys and set elements. Lists and dicts are NOT hashable.
String (str)
Text data, enclosed in quotes: "hello" or 'hello'.
Integer (int)
Whole numbers: 42, -7, 0.
Float (float)
Decimal numbers: 3.14, -0.5.
Boolean (bool)
Truth values: True or False.
f-string
String interpolation with f"..." and {} placeholders: f"Hello, {name}".
Function
A reusable block of code. Called with (): print("hi").
print()
Built-in function that outputs text to the console.
input()
Built-in function that reads a line of text from the user. Returns a string.
type()
Built-in function that returns the type of a value: type(42) → <class 'int'>.
REPL
Read-Eval-Print Loop. Interactive Python shell. Start with python in terminal.
Script
A .py file containing Python code, run with python filename.py.
Comment
Code ignored by Python. Starts with #. Used for notes.

Operator Concepts

Short-Circuit Evaluation
Logical and/or stop evaluating as soon as the result is determined. False and x never evaluates x. True or x never evaluates x.
Chained Comparisons
Python allows a < b < c instead of a < b and b < c. Each middle operand is evaluated only once.
Operator Precedence
The order in which Python evaluates operators in an expression. From highest: **, unary +-~, *///%, +-, shifts, bitwise, comparisons, not, and, or (lowest).
Operator Overloading
Operators like + behave differently based on operand types because types define special methods (e.g., __add__). int.__add__ does arithmetic; str.__add__ concatenates.
Right-Associativity
Some operators evaluate right-to-left. ** is right-associative: 2 ** 3 ** 2 = 2 ** (3 ** 2) = 2 ** 9 = 512.

Control Flow

if Statement
Runs a block only if a condition is truthy: if x > 0: print("positive"). Always ends with a colon.
elif
Short for "else if". Chains an additional condition after an if. Checked only if all previous conditions were false.
else
Fallback branch — runs when no if or elif condition was true. Optional, must be last.
Ternary Expression
One-line conditional: value_if_true if condition else value_if_false. Returns a value.
match/case
Structural pattern matching (Python 3.10+). Compares a subject against patterns. Supports literals, | for multiple patterns, _ as wildcard, and guards (if). More powerful than if/elif for complex matching.
Guard
An extra condition in a match/case pattern: case (x, y) if x == y:. Must be true for the case to match.

Loops

for Loop
Iterates over an iterable (list, string, tuple, etc.) one element at a time. Unlike C/JavaScript, does not use an index by default: for fruit in fruits: print(fruit).
while Loop
Repeats as long as a condition is truthy: while count < 5: count += 1. Use when number of iterations is unknown.
range()
Built-in function that generates an arithmetic progression of numbers. Lazy (memory-efficient). Forms: range(stop), range(start, stop), range(start, stop, step). The stop value is exclusive.
break
Exits the innermost enclosing for or while loop immediately.
continue
Skips the rest of the current loop iteration and moves to the next one.
Loop else
An optional else clause on for and while loops. Runs only if the loop completed without hitting a break. Does not run if loop was terminated by break, return, or an exception.
enumerate()
Built-in that adds a counter to an iterable. Yields (index, value) tuples. Preferred over range(len(seq)). Accepts a start parameter (default 0).
zip()
Built-in that pairs elements from multiple iterables, stopping at the shortest. Often used with for: for a, b in zip(list1, list2):.
dict.items()
Returns a view of key-value pairs. Used in loops: for k, v in person.items():. The preferred way to iterate over both keys and values.
dict.values()
Returns a view of values only. Used in loops: for v in person.values():. Preserves insertion order (Python 3.7+).
Set iteration
Looping over a set yields elements in arbitrary (unpredictable) order. Use sorted(set) for ordered iteration: for x in sorted(myset):.
Nested Loop
A loop inside another loop. The inner loop runs all its iterations for each iteration of the outer loop. Common for 2D data (matrices, grids).
Iterable
An object capable of returning its elements one at a time. Examples: list, string, tuple, dict, set, range, file handle. Can be used in a for loop or with iter().

Control Flow (continued)

pass
No-op statement. Does nothing. Used as a placeholder where syntax requires a statement. Also useful in loop bodies you haven't implemented yet.
