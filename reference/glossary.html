Python Glossary

Essential Python terminology. Updated as new concepts are learned.

Core Concepts

Variable
A name that references an object. Created with =: name = "Alice". Variables are labels on objects, not containers.
Object Identity (id())
Every object has a unique identity (memory address). id(x) returns it. Never changes for the object's lifetime.
is
Identity comparison. a is b is True only when a and b reference the exact same object. Different from == (value comparison).
Mutable
An object whose value can change after creation. Examples: list, dict, set.
Immutable
An object whose value cannot change after creation. Examples: int, float, str, tuple, bool.
Assignment
Binds a name to an object: x = 5. The right side is evaluated first, then the name is bound to the result.
Augmented Assignment
Combines operation and assignment: x += 1. For mutable objects, may operate in-place. Equivalent to x = x + 1 but x is evaluated only once.
Tuple Unpacking
Assigning multiple variables from an iterable: a, b, c = [1, 2, 3] or swapping: a, b = b, a.
Star Unpacking
Using * to capture remaining items: first, *rest = [1, 2, 3] → first=1, rest=[2, 3].
Type Annotation
Syntax for hinting variable types: name: str = "Alice". Ignored at runtime but used by linters and type checkers.
del
Removes a name binding (not the object itself). The object persists if other names reference it.
snake_case
Naming convention for Python variables and functions: lowercase with underscores between words. my_variable_name.
Truthiness
Every object's boolean value. False: None, False, zero, empty sequences/collections. Everything else is True.
Numeric Types
int (unlimited precision), float (double-precision), complex (real + imag). All immutable.
Sequence Types
Ordered collections with index access. list (mutable), tuple (immutable), range (immutable, memory-efficient).
Set Types
Unordered collections of unique, hashable elements. set (mutable), frozenset (immutable, hashable).
Mapping Type
dict — key-value pairs, insertion-ordered (3.7+), keys must be hashable.
Binary Types
bytes (immutable), bytearray (mutable), memoryview (zero-copy buffer access).
isinstance()
Checks if an object is an instance of a type: isinstance(x, int). Preferred over type(x) is int because it handles subclasses correctly.
Hashable
An object with a hash value that never changes. Immutable types are hashable. Required for dict keys and set elements. Lists and dicts are NOT hashable.
String (str)
Text data, enclosed in quotes: "hello" or 'hello'. Strings are sequences of Unicode code points, immutable, and hashable.
Integer (int)
Whole numbers: 42, -7, 0.
Float (float)
Decimal numbers: 3.14, -0.5.
Boolean (bool)
Truth values: True or False.
f-string
String interpolation with f"..." and {} placeholders: f"Hello, {name}".
Function
A reusable block of code. Called with (): print("hi").
print()
Built-in function that outputs text to the console.
input()
Built-in function that reads a line of text from the user. Returns a string.
type()
Built-in function that returns the type of a value: type(42) → <class 'int'>.
REPL
Read-Eval-Print Loop. Interactive Python shell. Start with python in terminal.
Script
A .py file containing Python code, run with python filename.py.
Comment
Code ignored by Python. Starts with #. Used for notes.

Operator Concepts

Short-Circuit Evaluation
Logical and/or stop evaluating as soon as the result is determined. False and x never evaluates x. True or x never evaluates x.
Chained Comparisons
Python allows a < b < c instead of a < b and b < c. Each middle operand is evaluated only once.
Operator Precedence
The order in which Python evaluates operators in an expression. From highest: **, unary +-~, *///%, +-, shifts, bitwise, comparisons, not, and, or (lowest).
Operator Overloading
Operators like + behave differently based on operand types because types define special methods (e.g., __add__). int.__add__ does arithmetic; str.__add__ concatenates.
Right-Associativity
Some operators evaluate right-to-left. ** is right-associative: 2 ** 3 ** 2 = 2 ** (3 ** 2) = 2 ** 9 = 512.

Control Flow

if Statement
Runs a block only if a condition is truthy: if x > 0: print("positive"). Always ends with a colon.
elif
Short for "else if". Chains an additional condition after an if. Checked only if all previous conditions were false.
else
Fallback branch — runs when no if or elif condition was true. Optional, must be last.
Ternary Expression
One-line conditional: value_if_true if condition else value_if_false. Returns a value.
match/case
Structural pattern matching (Python 3.10+). Compares a subject against patterns. Supports literals, | for multiple patterns, _ as wildcard, and guards (if). More powerful than if/elif for complex matching.
Guard
An extra condition in a match/case pattern: case (x, y) if x == y:. Must be true for the case to match.

Loops

for Loop
Iterates over an iterable (list, string, tuple, etc.) one element at a time. Unlike C/JavaScript, does not use an index by default: for fruit in fruits: print(fruit).
while Loop
Repeats as long as a condition is truthy: while count < 5: count += 1. Use when number of iterations is unknown.
range()
Built-in function that generates an arithmetic progression of numbers. Lazy (memory-efficient). Forms: range(stop), range(start, stop), range(start, stop, step). The stop value is exclusive.
break
Exits the innermost enclosing for or while loop immediately.
continue
Skips the rest of the current loop iteration and moves to the next one.
Loop else
An optional else clause on for and while loops. Runs only if the loop completed without hitting a break. Does not run if loop was terminated by break, return, or an exception.
enumerate()
Built-in that adds a counter to an iterable. Yields (index, value) tuples. Preferred over range(len(seq)). Accepts a start parameter (default 0).
zip()
Built-in that pairs elements from multiple iterables, stopping at the shortest. Often used with for: for a, b in zip(list1, list2):.
dict.items()
Returns a view of key-value pairs. Used in loops: for k, v in person.items():. The preferred way to iterate over both keys and values.
dict.values()
Returns a view of values only. Used in loops: for v in person.values():. Preserves insertion order (Python 3.7+).
Set iteration
Looping over a set yields elements in arbitrary (unpredictable) order. Use sorted(set) for ordered iteration: for x in sorted(myset):.
reversed()
Built-in that returns a reverse iterator over a sequence. Useful in loops: for x in reversed(seq):.
sorted()
Built-in that returns a new sorted list from an iterable. Leaves the original unchanged. Common in loops: for x in sorted(iterable):.
Loop variable scoping
Python's for loop variable leaks into the enclosing scope and retains its value after the loop ends. This differs from C/Java where the variable is local to the loop.
Nested Loop
A loop inside another loop. The inner loop runs all its iterations for each iteration of the outer loop. Common for 2D data (matrices, grids).
Iterable
An object capable of returning its elements one at a time. Examples: list, string, tuple, dict, set, range, file handle. Can be used in a for loop or with iter().

Control Flow (continued)

pass
No-op statement. Does nothing. Used as a placeholder where syntax requires a statement. Also useful in loop bodies you haven't implemented yet.

Strings

String indexing
Individual characters accessed by position: s[0] (first), s[-1] (last). Zero-based. Negative indices count from the end.
String slicing
Extract a substring: s[start:stop:step]. Stop is exclusive. All bounds are optional. Never raises an error — out-of-bounds indices are silently clamped. s[::-1] reverses the string.
String immutability
Strings cannot be modified in place. s[0] = "x" raises TypeError. All operations return a new string. This makes strings hashable (usable as dict keys / set elements).
.upper() / .lower()
Return a new string with all characters converted to uppercase/lowercase. "Hello".upper() → "HELLO".
.capitalize() / .title()
.capitalize() uppercases the first character only. .title() uppercases the first character of each word.
.swapcase()
Swaps uppercase ↔ lowercase: "Hello".swapcase() → "hELLO".
.casefold()
Aggressive lowercasing for caseless matching. More thorough than .lower() for non-ASCII text (e.g., German ß → ss).
.find() / .rfind()
Return the lowest/highest index where a substring is found. Returns -1 if not found. Accepts optional start and end range.
.index() / .rindex()
Like .find() / .rfind() but raises ValueError if substring is not found.
.count()
Returns the number of non-overlapping occurrences of a substring: "ababa".count("aba") → 1.
.replace()
Returns a copy with all occurrences of a substring replaced: "cat cat".replace("cat", "dog") → "dog dog". Optional count limits replacements.
.startswith() / .endswith()
Check if a string starts/ends with a given substring. Accepts a tuple of options: "file.txt".endswith((".txt", ".md")).
.removeprefix() / .removesuffix()
Python 3.9+. Remove a leading/trailing substring if present: "https://x.com".removeprefix("https://") → "x.com".
.split() / .rsplit()
Split a string into a list using a delimiter. Default splits on any whitespace (and discards empty strings). maxsplit limits the number of splits. rsplit splits from the right.
.join()
Concatenates an iterable of strings using the separator string: ", ".join(["a", "b", "c"]) → "a, b, c". Called on the separator, not the list. The idiomatic way to combine strings.
.partition() / .rpartition()
Split a string at the first/last occurrence of a separator. Returns a 3-tuple: (before, separator, after). Always returns three elements.
.splitlines()
Split on line boundaries (\n, \r\n, etc.). Optional keepends=True preserves the line break characters.
.strip() / .lstrip() / .rstrip()
Remove leading/trailing characters (default: whitespace). " hello ".strip() → "hello". Can specify chars to remove: "...hello...".strip(".") → "hello".
.center() / .ljust() / .rjust()
Pad a string to a given width. "hi".center(7) → " hi ". Optional fillchar (default space).
.zfill()
Left-pad with zeros to a given width. Sign-aware: "-5".zfill(5) → "-0005".
.expandtabs()
Replace tab characters (\t) with spaces, respecting tab stops. Default tab size is 8.
Classification methods (is*)
12 methods that check string content: .isalpha() (letters only), .isdigit() (digits only), .isalnum() (letters and digits), .isspace() (whitespace only), .islower() / .isupper() (case check), .istitle() (titlecased), .isascii() (all chars ASCII), .isidentifier() (valid Python identifier), .isprintable() (no control chars), .isdecimal() (decimal digits), .isnumeric() (any numeric including fractions/Roman). All return False for empty strings.
ord()
Built-in function that returns the Unicode code point (integer) of a character: ord("A") → 65, ord("😀") → 128512. Inverse of chr().
chr()
Built-in function that returns the character for a Unicode code point: chr(65) → "A". Inverse of ord(). Raises ValueError if the integer is out of valid Unicode range.
f-string format specifier
After the colon in an f-string expression: f"{value:width.precision type}". Examples: :.2f (2 decimal places), :>10 (right-align width 10), :^10 (center), :b (binary), :x (hex).
f-string debug (=)
Python 3.8+. f"{var=}" prints var=value. Useful for quick debugging: f"{name=} {age=}" → "name='Alice' age=30".
f-string conversion (!r, !s, !a)
!r applies repr(), !s applies str() (default), !a applies ascii() (escapes non-ASCII). f"{text!r}" → "'hello\\nworld'".
Raw string (r"...")
A string prefix that treats backslashes as literal characters. No escape sequences are processed. Essential for regular expressions (r"\d+\.\d+") and Windows paths (r"C:\Users\name"). Can combine with f-strings: rf"{value}".
Triple-quoted string
A string delimited by """ or ''' that spans multiple lines. Used for docstrings and long text blocks. Leading/trailing whitespace is preserved unless using textwrap.dedent().
Docstring
A triple-quoted string as the first statement in a function/class/module. Accessible via help() and obj.__doc__. Standard convention for documentation.
Unicode code point
An integer in the Unicode codespace, ranging from 0 to 0x10FFFF (1,114,111). Each character maps to a code point. Represented in Python with \u (16-bit), \U (32-bit), or \N{name} escapes.
.encode() / .decode()
.encode(encoding) converts a str to bytes. .decode(encoding) converts bytes to str. Common encodings: "utf-8" (default), "utf-16", "latin-1", "ascii". Error handling: "strict" (default, raises), "ignore", "replace" (adds ?), "xmlcharrefreplace".
bytes vs str
In Python 3, str holds Unicode text, bytes holds raw byte data. They are distinct types and cannot be mixed: "hello" + b" world" raises TypeError. Convert explicitly with .encode() / .decode() at program boundaries.
Unicode escape sequences
\u00e9 → é (16-bit), \U0001F600 → 😀 (32-bit), \N{GREEK SMALL LETTER PI} → π (named). Used in normal and raw strings to represent characters by their code point.
