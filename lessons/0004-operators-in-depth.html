Chained Comparisons

Python lets you chain comparisons naturally — this is a unique feature not all languages have:

x = 5
print(1 < x < 10)      # True  (same as: 1 < x and x < 10)
print(1 < x <= 5)      # True
print(10 > x > 0)       # True

Under the hood, a < b < c is equivalent to a < b and b < c — but b is evaluated only once.

Short-Circuit Evaluation

Python evaluates and/or lazily — it stops as soon as the result is determined:

def say(word):
    print(f"  ran say('{word}')")
    return word

print("and:")
result = say("a") and say("b")   # prints "ran say('a')" then stops (falsy)
# Only the first call runs because "a" is truthy but say returns "a",
# which is truthy — so it continues. Let's try with a falsy first value:

result = "" and say("b")          # "" is falsy, short-circuits — say("b") never runs
print(result)                     # ""  (returns the falsy value)

print("or:")
result = "hello" or say("b")     # "hello" is truthy, short-circuits — say("b") never runs

Key behavior: and and or return the actual value that determined the result, not necessarily True/False.

6. Bitwise Operators

These operate on integers at the binary-bit level. Less common in everyday code but essential for flags, permissions, and low-level work.

OperatorNameExampleBinaryResult
&AND5 & 3101 & 0111
|OR5 | 3101 | 0117
^XOR5 ^ 3101 ^ 0116
~NOT (invert)~5~101-6
<<Left shift5 << 1101 << 110
>>Right shift5 >> 1101 >> 12
# Bitwise AND — both bits must be 1
print(5 & 3)   # 1   (101 & 011 = 001)

# Bitwise OR — at least one bit is 1
print(5 | 3)   # 7   (101 | 011 = 111)

# Bitwise XOR — exactly one bit is 1
print(5 ^ 3)   # 6   (101 ^ 011 = 110)

# Left shift — shifts bits left (multiply by 2^n)
print(5 << 2)  # 20  (101 << 2 = 10100)

# Right shift — shifts bits right (floor-divide by 2^n)
print(20 >> 2) # 5   (10100 >> 2 = 101)

7. Operator Precedence

When multiple operators appear in one expression, Python follows a strict order. From highest to lowest precedence:

LevelOperatorsNotes
1 (highest)**Exponentiation (right-associative!)
2+x, -x, ~xUnary operators
3*, /, //, %Multiplication, division, floor div, modulo
4+, -Addition, subtraction
5<<, >>Bitwise shifts
6&Bitwise AND
7^Bitwise XOR
8|Bitwise OR
9==, !=, <, >, <=, >=, in, not in, is, is notComparisons, membership, identity
10notLogical NOT
11andLogical AND
12 (lowest)orLogical OR
# Without parentheses — uses precedence
result = 2 + 3 * 4 ** 2   # 2 + 3 * 16 = 2 + 48 = 50
print(result)

# With parentheses — explicit
result = ((2 + 3) * 4) ** 2  # (5 * 4) ** 2 = 20 ** 2 = 400
print(result)

When in doubt, use parentheses. They cost nothing and make intent obvious.

8. Operator Overloading (Brief Intro)

Operators work differently depending on the types involved because types define special methods (dunder methods) that implement each operator:

print("hello" + " world")  # str.__add__ — concatenation
print([1, 2] + [3])         # list.__add__ — list concatenation
print(3 + 4)                # int.__add__ — numeric addition

# Same operator (+), different behavior depending on type.

You can define these methods in your own classes to make custom objects support operators. We'll cover this in detail in the OOP phase.
