Lesson 9: Tuples — Python's Immutable Sequences

concept Tuples

Tuples are Python's immutable ordered sequence type. They are used for heterogeneous data that shouldn't change, for unpacking, as dictionary keys, and for function return values. This lesson covers everything from the official Python docs on tuples.

Primary sources: Python Docs — Tuples, Data Structures Tutorial, collections.namedtuple.

1. Tuple Basics

Tuples are ordered, heterogeneous sequences. The key thing to know: commas create tuples, not parentheses. Parentheses are often necessary for clarity, but the comma is what makes it a tuple:

# Commas create tuples
t = 1, 2, 3
print(t)          # (1, 2, 3)
print(type(t))    # 

# Parentheses are common but optional (except in ambiguity)
t2 = (1, 2, 3)    # same thing
print(t2)         # (1, 2, 3)

# Heterogeneous
mixed = (1, "hello", 3.14, True)
print(mixed)      # (1, 'hello', 3.14, True)

# Construction from any iterable
print(tuple("abc"))       # ('a', 'b', 'c')
print(tuple([1, 2, 3]))   # (1, 2, 3)
print(tuple(range(5)))    # (0, 1, 2, 3, 4)

# Nested tuples
nested = (1, (2, 3), (4, (5, 6)))
print(nested)             # (1, (2, 3), (4, (5, 6)))
print(nested[1])          # (2, 3)
print(nested[1][1])       # 3

2. Tuple Packing and Unpacking

The comma syntax is called tuple packing — multiple values are packed into a single tuple. The reverse is sequence unpacking (works on any sequence, not just tuples):

# Tuple packing: values → tuple
packed = 1, 2, 3
print(packed)  # (1, 2, 3)

# Tuple unpacking: tuple → values
a, b, c = packed
print(a, b, c)  # 1 2 3

# Swapping without a temp variable — the Pythonic way
x, y = 10, 20
x, y = y, x
print(x, y)     # 20 10

# Unpacking works on ANY sequence
a, b, c = [1, 2, 3]    # list
x, y = "hi"              # string
p, q, r = range(3)      # range
print(a, b, c)           # 1 2 3
print(x, y)              # h i
print(p, q, r)           # 0 1 2

# Starred unpacking — capture "rest" with *
first, *middle, last = range(10)
print(first)    # 0
print(middle)   # [1, 2, 3, 4, 5, 6, 7, 8]
print(last)     # 9

# Ignoring values with _
_, _, third = (10, 20, 30)
print(third)    # 30
💡 Multiple return values: When a function returns multiple values like return min_val, max_val, it's actually returning a single tuple that gets packed automatically. The caller can unpack it: lo, hi = find_min_max(data).

3. Single-Element and Empty Tuples

The trailing comma is required for a single-element tuple. Without it, parentheses are just for grouping:

# Empty tuple
empty = ()
print(type(empty))  # 
print(len(empty))   # 0

# Single-element tuple — trailing comma is MANDATORY
singleton = (1,)
print(type(singleton))  # 
print(singleton)        # (1,)

# Without comma — NOT a tuple, just the value
not_a_tuple = (1)
print(type(not_a_tuple))  # 

# Even without parentheses, the comma matters
another = 1,
print(type(another))  # 

# Common mistake: missing comma in a list of tuples
points = [(1, 2), (3, 4)]  # correct — list of two tuples
# points = [(1, 2), (3, 4)]  # wrong if you meant three items

4. Tuple Immutability (with nuance)

Tuples are immutable — you cannot change, add, or remove elements. However, if a tuple contains a mutable object (like a list), that object's contents CAN change:

# Tuples cannot be modified in place
t = (1, 2, 3)
# t[0] = 99      # TypeError: 'tuple' object does not support item assignment
# t.append(4)    # AttributeError: 'tuple' object has no attribute 'append'
# del t[0]       # TypeError: 'tuple' object doesn't support item deletion

# BUT: tuples can contain mutable objects
t = ([1, 2], [3, 4])
print(t)              # ([1, 2], [3, 4])
t[0].append(99)       # modify the list INSIDE the tuple
print(t)              # ([1, 2, 99], [3, 4]) — the tuple itself is unchanged!

# What does "immutable" mean for tuples?
# The TUPLE REFERENCE cannot change — it always points to the same objects.
# But those objects themselves might be mutable.

# Identity vs value
t1 = (1, 2, 3)
t2 = (1, 2, 3)
print(t1 == t2)   # True — same value
print(t1 is t2)   # False — different objects (usually)

# Concatenation creates a NEW tuple (immutable!)
t = (1, 2)
t = t + (3, 4)    # rebinds t to a new tuple object
print(t)          # (1, 2, 3, 4)
⚠️ The += illusion: t += (4,) looks like mutation but actually creates a new tuple and rebinds the variable. Unlike lists, += on tuples never operates in-place: 
t = (1, 2, 3)
id_before = id(t)
t += (4,)
print(id(t) == id_before)  # False — new object!

5. Tuple Methods — .count() and .index()

Tuples have only two methods (inherited from the immutable sequence protocol):

t = (1, 2, 3, 2, 4, 2, 5)

# .count() — how many times a value appears
print(t.count(2))       # 3
print(t.count(99))      # 0

# .index() — find first occurrence (raises ValueError if not found)
print(t.index(3))       # 2
print(t.index(2))       # 1 — first occurrence

# .index() with start/stop — search within a slice
print(t.index(2, 2))        # 3 — search starting at index 2
print(t.index(2, 2, 5))     # 3 — search between index 2 and 5

# Handle ValueError gracefully
try:
    idx = t.index(99)
except ValueError:
    print("99 not found in tuple")

6. Tuple Operations — +, *, in, len(), etc.

Tuples support all the common sequence operations, just like lists:

a = (1, 2, 3)
b = (4, 5, 6)

# Concatenation
print(a + b)        # (1, 2, 3, 4, 5, 6)

# Repetition
print(a * 3)        # (1, 2, 3, 1, 2, 3, 1, 2, 3)

# Membership
print(2 in a)       # True
print(99 not in a)  # True

# Length
print(len(a))       # 3

# Min / Max / Sum (works on homogeneous numeric tuples)
numbers = (3, 1, 4, 1, 5, 9)
print(min(numbers))     # 1
print(max(numbers))     # 9
print(sum(numbers))     # 23

# sorted() returns a list (not a tuple)
print(sorted(numbers))  # [1, 1, 3, 4, 5, 9]

# Indexing and slicing (same as strings/lists)
print(a[0])          # 1
print(a[-1])         # 3
print(a[1:3])        # (2, 3)
print(a[::-1])       # (3, 2, 1)
⚠️ Concatenation performance: Building up a tuple with repeated + in a loop has quadratic runtime cost (each iteration creates a new tuple). If you need to build a tuple incrementally, build a list first and convert at the end: 
# BAD — O(n²)
result = ()
for x in range(1000):
    result += (x,)

# GOOD — O(n)
result = []
for x in range(1000):
    result.append(x)
result = tuple(result)

7. Lexicographic Comparison

Tuples (and lists) are compared lexicographically — element by element from left to right. This is extremely useful for sorting:

# Element-by-element comparison
print((1, 2, 3) < (1, 2, 4))      # True — first difference at index 2
print((1, 2, 3) < (1, 2, 3, 0))   # True — shorter sequence < longer (if all equal)
print((1, 2) < (1, 2, 3))         # True
print((1, 2, 3) == (1.0, 2.0, 3.0))  # True — values compared, not types

# Nested tuples are compared recursively
print((1, (2, 3)) < (1, (2, 4)))  # True

# Sorting a list of tuples sorts by first element, then second, etc.
coordinates = [(3, 1), (1, 4), (1, 2), (2, 3)]
coordinates.sort()
print(coordinates)  # [(1, 2), (1, 4), (2, 3), (3, 1)]

# You can sort by any element using key=
points = [(4, 2), (1, 5), (3, 3)]
points.sort(key=lambda p: p[1])  # sort by second element
print(points)  # [(4, 2), (3, 3), (1, 5)]

# Comparing different types raises TypeError (usually)
# print((1, 2) < (1, "hello"))  # TypeError: '<' not supported between str and int
💡 Multiple criteria sorting: Because tuples compare element by element, you can create composite sort keys. For example, sorted(people, key=lambda p: (p.age, p.name)) sorts by age first, then by name for equal ages.

8. Hashability — Tuples as Dict Keys and Set Members

Tuples are hashable if (and only if) all their elements are hashable. This makes them usable as dictionary keys and set members — something lists cannot do:

# Tuples as dictionary keys — composite keys
grades = {
    ("Alice", "Math"): 95,
    ("Alice", "Science"): 88,
    ("Bob", "Math"): 92,
}

print(grades[("Alice", "Math")])          # 95

# Iterate over composite-key dict
for (student, subject), score in grades.items():
    print(f"{student} got {score} in {subject}")

# Tuples as set members
valid_codes = {(200, "OK"), (404, "Not Found"), (500, "Server Error")}
print((404, "Not Found") in valid_codes)  # True

# TUPLES with mutable elements are NOT hashable
# t = ([1, 2], 3)  # unhashable type: 'list'
# d = {t: "value"}  # TypeError!

# LISTS cannot be dict keys at all
# d = {[1, 2]: "value"}  # TypeError: unhashable type: 'list'
💡 Why can tuples be dict keys but not lists? Dictionary keys must be hashable, which requires the object's hash value to never change. Lists are mutable — their contents can change, so their hash would too. Tuples are immutable, so their hash is stable (provided all elements are hashable). This is the same reason tuples are immutable in the first place.

9. Named Tuples — collections.namedtuple

Named tuples assign meaning to each position. They are regular tuples (indexable, iterable, unpackable) plus attribute access by field name:

from collections import namedtuple

# Define a named tuple type
Point = namedtuple("Point", ["x", "y"])

# Create instances — positional or keyword args
p1 = Point(3, 4)
p2 = Point(x=1, y=2)

# Access by index (like a regular tuple)
print(p1[0])            # 3
print(p1[1])            # 4

# Access by attribute name
print(p1.x)             # 3
print(p1.y)             # 4

# Unpack like a regular tuple
x, y = p1
print(x, y)             # 3 4

# Nice repr
print(p1)               # Point(x=3, y=4)

# _make(iterable) — create from a sequence
data = [5, 6]
p3 = Point._make(data)
print(p3)               # Point(x=5, y=6)

# _asdict() — convert to an ordered dict
d = p1._asdict()
print(d)                # {'x': 3, 'y': 4}

# _replace(**kwargs) — create a NEW tuple with changed fields
p4 = p1._replace(x=99)
print(p1)               # Point(x=3, y=4) — original unchanged
print(p4)               # Point(x=99, y=4)

# _fields — introspect field names
print(Point._fields)    # ('x', 'y')

# Default values — applied to rightmost fields
Student = namedtuple("Student", ["name", "grade", "age"], defaults=["", 0])
s = Student("Alice")    # name="Alice", grade="", age=0
print(s)                # Student(name='Alice', grade='', age=0)

# Rename invalid field names
Node = namedtuple("Node", ["id", "class", "def"], rename=True)
print(Node._fields)     # ('id', '_1', '_2') — reserved words renamed
n = Node(1, 2, 3)
print(n._1)             # 2
💡 When to use namedtuple: Use named tuples when you want lightweight, immutable data containers with named fields — they take no more memory than regular tuples. For type safety with type hints, see typing.NamedTuple. For mutable data containers, consider dataclasses.

10. When to Use Tuples

Tuples and lists serve different purposes in idiomatic Python:

# 1. Multiple return values (always a tuple)
def min_max(numbers):
    return min(numbers), max(numbers)

lo, hi = min_max([3, 1, 4, 1, 5, 9])
print(lo, hi)           # 1 9

# 2. Function argument packing/unpacking
def print_point(x, y):
    print(f"({x}, {y})")

coords = (3, 4)
print_point(*coords)     # (3, 4)

# 3. Data integrity — record shouldn't change
person = ("Alice", "alice@email.com", 30)
# Tuples signal "this data is a fixed record" to readers

# 4. Heterogeneous data (different types, related values)
city_data = ("London", 8_982_000, 51.5, -0.1)

# 5. Unpacking-heavy code — zip, enumerate, dict.items()
names = ["Alice", "Bob", "Charlie"]
scores = [95, 87, 92]
for i, (name, score) in enumerate(zip(names, scores)):
    print(f"{i}: {name} = {score}")

# 6. Memory efficiency (slightly smaller than lists)
from sys import getsizeof
print(getsizeof((1, 2, 3)))    # typically 64 bytes
print(getsizeof([1, 2, 3]))    # typically 88 bytes (or more)
