Lesson 5: Conditionals — Making Decisions

concept Conditionals

Programs need to make decisions. Python's if/elif/else statements let you run different code depending on conditions. Since Python 3.10 there's also match/case for pattern matching.

Primary source: Python Docs — More Control Flow Tools

1. The if Statement

The simplest conditional runs a block only when a condition is true:

x = 10
if x > 5:
    print("x is greater than 5")

Key rules:

  • The condition must end with a colon (:)
  • The body must be indented (4 spaces by convention)
  • Any expression that evaluates to a boolean works — Python checks truthiness
if 42:              # truthy — always runs
    print("yes")

if []:              # falsy — never runs
    print("no")

2. elif and else

Chain multiple conditions with elif (short for "else if") and provide a fallback with else:

score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"

print(grade)  # B

Rules:

  • Conditions are checked top to bottom
  • The first true condition runs; the rest are skipped
  • elif can appear zero or more times
  • else is optional and must be last
  • Only one branch ever executes
Note: Unlike some languages (JavaScript, C), there is no switch statement. Use if/elif/else or the newer match/case (see section 6).

3. Truthiness in Conditions (Review)

Every Python object has a truth value. These are always falsy:

  • None
  • False
  • Zero in any numeric type: 0, 0.0, 0j
  • Empty sequences/collections: "", [], (), {}, set(), range(0)

Everything else is truthy.

Idiomatic Python uses implicit truthiness rather than comparing to True or False:

# Good — Pythonic
if items:
    print(f"Got {len(items)} items")

# Unnecessary — don't do this
if len(items) > 0:
    print(f"Got {len(items)} items")

# Checking for None explicitly is fine (and required)
if result is None:
    print("No result")

4. Nested Conditionals

You can put an if inside another if. Nested conditionals let you express more complex logic:

age = 25
has_id = True

if age >= 18:
    if has_id:
        print("You can enter")
    else:
        print("Need ID")
else:
    print("Too young")

Deep nesting (3+ levels) is a code smell — consider refactoring or using and/or instead:

# Instead of nesting:
if age >= 18 and has_id:
    print("You can enter")
elif age >= 18:
    print("Need ID")
else:
    print("Too young")

5. Ternary Expression (Conditional Expression)

Python has a one-line conditional expression: x if condition else y. It returns a value:

age = 20
status = "adult" if age >= 18 else "minor"
print(status)  # adult

# Equivalent to:
if age >= 18:
    status = "adult"
else:
    status = "minor"

Use judiciously — great for simple assignments, bad for complex logic:

# Good use
discount = 0.1 if is_member else 0.0

# Bad use — hard to read
result = "positive" if x > 0 else ("zero" if x == 0 else "negative")

6. match/case (Structural Pattern Matching)

Added in Python 3.10, match/case is a more powerful alternative to long if/elif chains. It compares a subject value against patterns:

status_code = 404

match status_code:
    case 200:
        print("OK")
    case 301 | 302:
        print("Redirect")
    case 400:
        print("Bad Request")
    case 404:
        print("Not Found")
    case _:
        print("Unknown")

Key points:

  • _ is the wildcard — matches anything (like default in other languages)
  • | combines multiple patterns in one case
  • Only the first matching case runs
  • No fall-through (unlike C/JavaScript switch)

You can also unpack sequences and bind variables:

point = (5, 0)

match point:
    case (0, 0):
        print("Origin")
    case (0, y):
        print(f"On y-axis at y={y}")
    case (x, 0):
        print(f"On x-axis at x={x}")
    case (x, y):
        print(f"At ({x}, {y})")

You can add a guard (an extra condition) with if:

match point:
    case (x, y) if x == y:
        print(f"On diagonal at ({x}, {y})")
    case (x, y):
        print(f"Not on diagonal")

We'll revisit match/case in later lessons when we cover classes and data structures.

7. pass — Do Nothing

pass is a placeholder that does nothing. Use it when syntax requires a statement but you haven't written the code yet:

if condition:
    pass  # TODO: implement later

while True:
    pass  # busy-wait

Exercises

Create each in exercises/05-conditionals/.

Exercise 1: Number Classifier

Write a program that asks the user for a number and prints whether it's positive, negative, or zero. Handle the case where the input is not a valid number with a try/except.

Enter a number: 5
5 is positive.

Enter a number: -3
-3 is negative.

Enter a number: 0
The number is zero.

Enter a number: abc
That's not a valid number.

Exercise 2: Even or Odd + Divisibility

Ask the user for a number. Print whether it's even or odd, and whether it's divisible by 5 and/or 3. Use the % operator.

Enter a number: 15
15 is odd.
15 is divisible by 5 and 3.

Enter a number: 10
10 is even.
10 is divisible by 5 but not 3.

Exercise 3: Vowel or Consonant

Ask the user for a single letter. Use a match/case statement with the | operator to print whether it's a vowel or consonant. Handle uppercase and lowercase. If it's not a letter at all, print an error.

Enter a letter: a
a is a vowel.

Enter a letter: B
B is a consonant.

Enter a letter: 5
That's not a letter.

Exercise 4: Ternary — Discount Calculator

Ask the user for the purchase amount and whether they're a member ("yes" or "no"). Use a ternary expression to set the discount: 15% for members, 5% for non-members. Print the final price.

Enter amount: 200
Are you a member? (yes/no): yes
Final price: $170.00

Enter amount: 200
Are you a member? (yes/no): no
Final price: $190.00

Exercise 5: Shipping Calculator

Write a program that calculates shipping cost based on order total and distance:

  • Orders over $50 ship free
  • Orders $25-$50: $5 flat + $0.50 per km
  • Orders under $25: $3 flat + $1.00 per km

If the shipping cost exceeds the order total, print "Shipping is more than the item value!" instead of the cost.

Enter order total: 100
Shipping is free! Total: $100.00

Enter order total: 30
Enter distance (km): 10
Shipping cost: $10.00. Total: $40.00

Enter order total: 15
Enter distance (km): 20
Shipping is more than the item value!

Exercise 6: Simple Calculator with match/case

Revisit Exercise 1 from Lesson 4 (Arithmetic Calculator). This time, implement the operator selection using match/case instead of if/elif. Use | to combine /, //, % into one case for the division-by-zero check.
