Lesson 7: Strings in Depth — Text as a First-Class Citizen

concept Strings

You've used strings since Lesson 1 — but Python's str type is surprisingly deep. Strings are sequences of Unicode code points, they support powerful slicing and querying operations, and they come with dozens of built-in methods. This lesson covers everything you need to handle text effectively.

Primary source: Python Docs — Text Sequence Type str and Fancier Output Formatting.

1. String Indexing and Slicing

Each character in a string has a position (index). Python uses zero-based indexing, and supports negative indices counting from the end:

s = "Python"
#     012345   (positive indices)
#    -6-5-4-3-2-1  (negative indices)

print(s[0])    # P
print(s[-1])   # n
print(s[2])    # t

Slicing extracts a substring: [start:stop:step]. The stop index is exclusive:

s = "Python"

print(s[0:3])    # Pyt   (indices 0, 1, 2)
print(s[:3])     # Pyt   (start defaults to 0)
print(s[3:])     # hon   (stop defaults to end)
print(s[-3:])    # hon   (last 3 characters)
print(s[::2])    # Pto   (every 2nd character)
print(s[::-1])   # nohtyP (reverse via step=-1)
print(s[1:5:2])  # yh    (indices 1, 3)
⚠️ Slicing never raises an error. If start or stop is out of bounds, Python silently clamps to the string length. This is different from index access (s[100] raises IndexError).

2. String Immutability

Strings are immutable — you cannot change a character in place:

s = "hello"
s[0] = "H"  # TypeError: 'str' object does not support item assignment

All string operations return a new string rather than modifying the original:

s = "hello"
upper_s = s.upper()
print(s)       # hello (unchanged)
print(upper_s) # HELLO (new string)

When you reassign a variable, you're pointing it at a new string object, not modifying the old one:

name = "Alice"
name = name.upper()  # name now points to "ALICE"
# The original "Alice" string still exists but is unreachable.

Because of immutability, strings are hashable — they can be used as dictionary keys and set elements.

3. Case Methods

Python provides several methods for changing or checking letter case:

text = "hello World"

print(text.upper())        # HELLO WORLD
print(text.lower())        # hello world
print(text.capitalize())   # Hello world (first letter uppercase, rest lowercase)
print(text.title())        # Hello World (each word's first letter uppercase)
print(text.swapcase())     # HELLO wORLD (swap uppercase ↔ lowercase)

# casefold() is more aggressive than lower() — used for caseless matching
german = "Straße"
print(german.lower())      # straße
print(german.casefold())   # strasse
💡 When to use each:
  • .lower() — general case-insensitive comparison
  • .casefold() — when dealing with non-ASCII text (German ß, Turkish İ, etc.)
  • .title() — formatting names and titles (but watch for edge cases like "O'Brien")
  • .capitalize() — formatting sentences

4. Search and Replace Methods

Finding substrings within a string:

text = "the cat in the hat"

print(text.find("cat"))    # 4 (lowest index where "cat" starts)
print(text.find("dog"))    # -1 (not found)
print(text.rfind("the"))   # 11 (highest index where "the" starts)

# .index() is like .find() but raises ValueError on failure
print(text.index("cat"))   # 4
# text.index("dog")        # ValueError: substring not found

print(text.count("the"))   # 2 (non-overlapping occurrences)

# Check start and end
print(text.startswith("the"))   # True
print(text.endswith("hat"))     # True

# Modern Python (3.9+): removeprefix and removesuffix
url = "https://example.com"
print(url.removeprefix("https://"))  # example.com
print(url.removesuffix(".com"))      # https://example

# Replace substrings
print(text.replace("cat", "dog"))  # the dog in the hat
print(text.replace("the", "a"))    # a cat in a hat
# .replace() with count argument:
print(text.replace("the", "a", 1)) # a cat in the hat (only first occurrence)

5. Split and Join

Splitting turns a string into a list of substrings. Joining does the reverse:

# Split — default splits on any whitespace
sentence = "the quick brown fox"
words = sentence.split()
print(words)  # ['the', 'quick', 'brown', 'fox']

# Split with a specific delimiter
csv_line = "Alice,25,Engineer"
fields = csv_line.split(",")
print(fields)  # ['Alice', '25', 'Engineer']

# Limit splits with maxsplit
data = "one two three four"
print(data.split(maxsplit=2))     # ['one', 'two', 'three four']
print(data.rsplit(maxsplit=2))    # ['one two', 'three', 'four']

# Partition — split at first occurrence, returns 3-tuple
print("a-b-c".partition("-"))     # ('a', '-', 'b-c')
print("a-b-c".rpartition("-"))    # ('a-b', '-', 'c')

# Splitlines — split on newline boundaries
multiline = "line1\nline2\r\nline3"
print(multiline.splitlines())     # ['line1', 'line2', 'line3']

# Join — the inverse of split
result = " | ".join(fields)
print(result)  # Alice | 25 | Engineer

# Join is called ON the separator string, not on the list
parts = ["2026", "07", "04"]
print("-".join(parts))            # 2026-07-04

# Join works with any iterable of strings
print("".join(["a", "b", "c"]))   # abc
⚠️ Join pitfall: ", ".join(items) joins items with a comma-space between them. There is no trailing separator. This is the most common string-combining pattern in Python.

6. Strip and Pad Methods

Stripping removes unwanted characters from edges. Padding adds characters to meet a width:

# Stripping — default removes whitespace
messy = "  \t  hello   \n"
print(repr(messy.strip()))        # 'hello'
print(repr(messy.lstrip()))       # 'hello   \n'
print(repr(messy.rstrip()))       # '  \t  hello'

# Strip specific characters (not just whitespace)
url = "www.example.com"
print(url.strip("w.com"))         # example

# Padding
print("42".center(10))            # '    42    '
print("42".ljust(10))             # '42        '
print("42".rjust(10))             # '        42'
print("42".zfill(10))             # '0000000042' (zeros, sign-aware)
print("-42".zfill(10))            # '-000000042'

# Expand tabs to spaces
print("a\tb".expandtabs(4))       # 'a   b'

7. Classification (is*) Methods

Python provides 12 methods to check what a string is made of:

def classify(s):
    print(f"{s!r:20} isalpha={s.isalpha():5} isdigit={s.isdigit():5} "
          f"isalnum={s.isalnum():5} isspace={s.isspace():5} "
          f"islower={s.islower():5} isupper={s.isupper():5}")

classify("Hello")   # alpha, not digit, alnum, not space, mixed case
classify("42")      # not alpha, digit, alnum, not space
classify("abc123")  # not pure alpha nor pure digit, but isalnum
classify("   ")     # not alpha, not digit, not alnum, is space
classify("hello")   # all lower
classify("HELLO")   # all upper

# More specialized checks
print("42".isdecimal())    # True  (decimal digits)
print("½".isdecimal())     # False (fraction)
print("½".isnumeric())     # True  (any numeric)
print("Ⅳ".isnumeric())     # True  (Roman numeral)

print(" ".isprintable())   # True  (space is printable)
print("\n".isprintable())  # False (newline is not printable)

print("hello".isidentifier())  # True
print("2cool".isidentifier())  # False (starts with digit)

print("".isascii())        # True  (empty string returns True)
print("café".isascii())    # False (é is non-ASCII)

8. ord() and chr()

Every character in Python has an integer code point (its Unicode number). ord() converts character → integer, chr() converts integer → character:

print(ord("A"))    # 65
print(ord("a"))    # 97
print(ord("0"))    # 48
print(ord("é"))    # 233
print(ord("😀"))   # 128512

print(chr(65))     # A
print(chr(97))     # a
print(chr(128512)) # 😀

# Useful for shifting characters (Caesar cipher)
def shift_char(c, amount):
    if c.isalpha():
        base = ord("A") if c.isupper() else ord("a")
        return chr((ord(c) - base + amount) % 26 + base)
    return c

# Letter positions relative to their case base
print(ord("C") - ord("A"))  # 2 (C is the 3rd letter, zero-indexed)
print(ord("z") - ord("a"))  # 25 (z is the last letter)
💡 Code points vs bytes: ord() returns the Unicode code point, not the byte value. ASCII characters (0–127) happen to be one byte in UTF-8, but characters like é (code point 233) take 2 bytes in UTF-8, and emoji like 😀 (code point 128512) take 4 bytes.

9. Advanced f-strings

You've used basic f-strings: f"Hello, {name}". They can do much more:

# Format specifiers (after a colon)
value = 12.3456
print(f"{value:.2f}")      # 12.35  (2 decimal places)
print(f"{value:10.2f}")    # '     12.35' (width 10, right-aligned)
print(f"{value:<10.2f}")   # '12.35     ' (left-aligned)
print(f"{value:^10.2f}")   # '  12.35   ' (centered)
print(f"{value:010.2f}")   # 000012.35   (zero-padded)

# Integer formatting
n = 42
print(f"{n:b}")            # 101010  (binary)
print(f"{n:o}")            # 52      (octal)
print(f"{n:x}")            # 2a      (hex lowercase)
print(f"{n:X}")            # 2A      (hex uppercase)

# Debug output — Python 3.8+
name = "Alice"
age = 30
print(f"{name=}, {age=}")  # name='Alice', age=30
print(f"{name=} {age=}")   # name='Alice' age=30

# Conversions
text = "hello\nworld"
print(f"{text!r}")         # 'hello\nworld'  (repr)
print(f"{text!s}")         # hello\nworld    (str, default)
print(f"{text!a}")         # 'hello\\nworld' (ascii, escapes non-ASCII)

# Nested expressions (Python 3.12+ allows backslashes in f-strings)
width = 10
precision = 3
print(f"{value:{width}.{precision}f}")  # '        12.346'

# Call methods inside f-strings
print(f"{name.upper()} is {age}")       # ALICE is 30

10. Raw Strings

Raw strings (prefix r) treat backslashes as literal characters — no escape sequences are processed:

# Normal string — backslash escapes are interpreted
normal = "C:\Users\name"
print(normal)  # C:\Users\ame (\n becomes newline!) — bug!

# Raw string — backslashes are literal
raw = r"C:\Users\name"
print(raw)     # C:\Users\name  (correct!)

# Common use case: regular expressions
import re
pattern = r"\d+\.\d+"  # matches numbers like 3.14
# Without raw string you'd need "\\d+\\.\\d+"

# Raw strings still end with a backslash if that's the last char
# This causes a syntax error:
# bad = r"\"       # SyntaxError
# Use a normal string or concatenation instead:
ok = r"path" "\\"  # 'path\\'

Raw strings work with all string prefixes: r"", rf"" (raw f-string), rb"" (raw bytes).

11. Multi-line Strings

Triple quotes (""" or ''') create strings that span multiple lines:

# Triple double quotes
poem = """Roses are red,
Violets are blue,
Python is fun,
And so are you."""

print(poem)
# Roses are red,
# Violets are blue,
# Python is fun,
# And so are you.

# Triple single quotes work the same way
message = '''This is a
multi-line
message.'''

# Explicit newline with backslash continuation (within regular strings)
long = "This is a very long string that we want to \
break across two lines in the source code."
# The backslash continues the string — no newline in the output

# Removing indentation — textwrap.dedent
import textwrap
code = textwrap.dedent("""\
    def hello():
        print("hi")
""")
# The leading indentation is stripped relative to the first line
💡 Triple quotes for docstrings: When a triple-quoted string is the first statement in a function, class, or module, it becomes its docstring — used by help() and documentation tools. You'll see this in every Python codebase.

12. Unicode and Encoding

Python strings are sequences of Unicode code points (not bytes). Encoding converts a string to bytes for storage/transmission; decoding converts bytes back to a string:

# Encode str → bytes
text = "café"
utf8_bytes = text.encode("utf-8")
print(utf8_bytes)         # b'caf\xc3\xa9'
print(type(utf8_bytes))   # <class 'bytes'>

# Decode bytes → str
decoded = utf8_bytes.decode("utf-8")
print(decoded)            # café

# Other encodings
print(text.encode("utf-16"))   # b'\xff\xfec\x00a\x00f\x00\xe9\x00'
print(text.encode("latin-1"))  # b'caf\xe9' (1 byte per char, limited)

# Error handling
# print(text.encode("ascii"))  # UnicodeEncodeError: 'é' not in ASCII
print(text.encode("ascii", errors="ignore"))        # b'caf'
print(text.encode("ascii", errors="replace"))       # b'caf?'
print(text.encode("ascii", errors="xmlcharrefreplace"))  # b'café'

# Unicode escape sequences
print("\u00e9")          # é (16-bit Unicode)
print("\U0001F600")      # 😀 (32-bit Unicode, needs 8 hex digits)
print("\N{GREEK SMALL LETTER PI}")  # π (named Unicode escape)

# Unicode identifiers — Python allows Unicode in variable names
π = 3.14159
print(π)                 # 3.14159
⚠️ The bytes vs str distinction matters: In Python 3, strings and bytes are different types. You cannot mix them: 
"hello" + b" world"  # TypeError: can only concatenate str (not "bytes") to str
Always encode/decode explicitly at the boundaries of your program (file I/O, network, database).
