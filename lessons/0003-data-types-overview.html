Lesson 3: Python Data Types — The Full Landscape

concept Data Types

In Lesson 1 you met the four basic types (int, float, str, bool). But Python has many more built-in types. This lesson gives you the bird's-eye view — all the types, how they're organized, and when to use each. Later lessons will deep-dive into each one.

Primary source: Python Docs — Built-in Types

1. The Python Type Hierarchy

Everything in Python is an object. Every object has a type. Types determine what you can do with an object.

print(type(42))         # <class 'int'>
print(type("hello"))    # <class 'str'>
print(type([1, 2]))     # <class 'list'>
print(type(print))      # <class 'builtin_function_or_method'>

Here's the full landscape of built-in types, grouped by category:

2. Numeric Types: int, float, complex

Numbers are the simplest types. All are immutable.

# int — unlimited precision
big = 2 ** 1000
print(big)  # 1071508607186267... — no overflow!

# float — double precision (like C double)
pi = 3.14159

# complex — two floats (real + imaginary)
z = 3 + 5j
print(z.real)  # 3.0
print(z.imag)  # 5.0

See docs.

3. Boolean: bool

bool is a subtype of int. True == 1, False == 0.

print(True == 1)   # True
print(False == 0)  # True
print(True + 2)    # 3 (but don't do this!)

# Use bool() to check truthiness of any value
print(bool(1))     # True
print(bool(0))     # False
print(bool(""))    # False
print(bool("hi"))  # True
print(bool([]))    # False

See docs.

4. Truthiness: Every Object Is True or False

Every Python object can be used in a boolean context. The following are always false:

  • None
  • False
  • Zero of any numeric type: 0, 0.0, 0j
  • Empty sequences/collections: "", [], (), {}, set(), range(0)

Everything else is True. This is defined by __bool__() or __len__() methods (see docs).

5. Sequence Types: list, tuple, range

Sequences are ordered collections you can index into with [i].

# list — mutable, heterogeneous
fruits = ["apple", "banana", "cherry"]
fruits.append("date")
print(fruits[1])     # banana

# tuple — immutable, heterogeneous
point = (3, 4)
# point[0] = 5       # TypeError!

# range — immutable, memory-efficient sequence of numbers
r = range(0, 10, 2)
print(list(r))       # [0, 2, 4, 6, 8]
print(5 in r)        # False (tested without building the list)

All sequences share common operations: indexing, slicing, len(), in, +, *, count(), index() (see docs).

Notable: range uses constant memory — it stores only start, stop, step, not all values.

6. Text Type: str

Strings are immutable sequences of Unicode code points (characters). Not bytes — Unicode.

text = "Hello, 世界"  # Unicode supported natively
print(len(text))       # 9 (characters, not bytes)
print(text[7])         # '世'

Strings have a huge set of methods: .upper(), .split(), .join(), .strip(), .replace(), etc. We'll cover these in the dedicated str lesson.

7. Binary Types: bytes, bytearray, memoryview

These handle raw binary data (bytes, not text).

  • bytes — immutable sequence of bytes (0-255)
  • bytearray — mutable version of bytes
  • memoryview — access a buffer without copying
b = b"hello"         # bytes literal
ba = bytearray(b)     # mutable copy
ba[0] = 72            # change 'h' to 'H' (ASCII 72)
print(ba)             # bytearray(b'Hello')

8. Set Types: set, frozenset

Sets are unordered collections of unique, hashable elements.

# set — mutable
s = {1, 2, 3, 1, 2}
print(s)          # {1, 2, 3} — duplicates removed!
s.add(4)

# frozenset — immutable, can be used as dict key
fs = frozenset([1, 2, 3])

Fast membership testing: 1000000 in s is O(1) average. See docs.

9. Mapping Type: dict

Dictionaries store key-value pairs. Keys must be hashable (immutable types).

ages = {"Alice": 30, "Bob": 25}
ages["Charlie"] = 35
print(ages["Alice"])   # 30
print("Bob" in ages)   # True (fast lookup)

Since Python 3.7, dicts preserve insertion order. See docs.

10. NoneType — The Null Object

None is a singleton representing the absence of a value. It's False in boolean context.

result = None
if result is None:         # always use "is None", not "== None"
    print("no result yet")

11. Type Conversion

You can convert between types using constructor functions:

int("42")        # 42    str → int
float("3.14")    # 3.14  str → float
str(42)          # "42"  int → str
list("hello")    # ['h', 'e', 'l', 'l', 'o']
tuple([1, 2])    # (1, 2)
set([1, 2, 2])   # {1, 2} — deduplication!
bool(1)          # True
bool(0)          # False

12. How to Check and Compare Types

x = 42

# Check type
print(type(x))           # <class 'int'>
print(type(x) is int)    # True

# Check if instance of a type (preferred)
print(isinstance(x, int))                # True
print(isinstance(x, (int, float)))       # True (checks against multiple)

# Duck typing — "if it walks like a duck..."
# Prefer isinstance() over type() for most code
