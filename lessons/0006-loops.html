Lesson 6: Loops — Repeating Work

concept Loops

Programs often need to repeat work — process every item in a list, keep asking until valid input is given, or compute a running total. Python gives you two looping tools: for (iterate over a sequence) and while (repeat while a condition is true).

Primary source: Python Docs — More Control Flow Tools (sections 4.2–4.5, 4.6) and Looping Techniques (section 5.6).

1. The for Loop

Python's for loop iterates over any iterable — a sequence like a list, string, or tuple. Unlike C or JavaScript, it doesn't use an index variable by default:

fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)

# apple
# banana
# cherry

You can also iterate directly over a string (character by character):

for ch in "hello":
    print(ch, end=" ")
# h e l l o

Key point: the loop variable (fruit) takes on each value in order. The loop body runs once per element.

⚠️ Modifying a list while iterating over it is tricky. If you need to remove items, iterate over a copy instead: 
for item in items.copy():
    if condition(item):
        items.remove(item)
Or build a new list with a comprehension (see Lesson 13).

2. range() — Generating Number Sequences

range() produces an arithmetic progression of numbers. It's lazy (memory-efficient) — it yields values one at a time, not a whole list:

for i in range(5):
    print(i, end=" ")
# 0 1 2 3 4

Common forms:

range(stop)        # 0 to stop-1
range(start, stop) # start to stop-1
range(start, stop, step) # step can be negative
list(range(5, 10))       # [5, 6, 7, 8, 9]
list(range(0, 10, 3))    # [0, 3, 6, 9]
list(range(-10, -100, -30))  # [-10, -40, -70]

range() is often used when you need an index — but prefer enumerate() (see section 6) for cleaner code:

words = ["cat", "window", "defenestrate"]
for i in range(len(words)):
    print(i, words[i])

# 0 cat
# 1 window
# 2 defenestrate

3. The while Loop

A while loop repeats as long as its condition is truthy. Use it when you don't know the number of iterations in advance:

count = 0
while count < 3:
    print(f"Count is {count}")
    count += 1

# Count is 0
# Count is 1
# Count is 2

Classic use case: keep prompting until valid input:

while True:
    reply = input("Type 'quit' to exit: ")
    if reply == "quit":
        break
    print(f"You said: {reply}")
⚠️ Infinite loop risk! If the condition never becomes false, the loop runs forever. Use break or ensure the condition eventually changes.

4. break and continue

break exits the innermost loop immediately:

for n in range(2, 10):
    for x in range(2, n):
        if n % x == 0:
            print(f"{n} = {x} * {n//x}")
            break

# 4 = 2 * 2
# 6 = 2 * 3
# 8 = 2 * 4
# 9 = 3 * 3

continue skips the rest of the current iteration and moves to the next one:

for num in range(2, 10):
    if num % 2 == 0:
        print(f"Even: {num}")
        continue
    print(f"Odd: {num}")

# Even: 2
# Odd: 3
# Even: 4
# Odd: 5
# Even: 6
# Odd: 7
# Even: 8
# Odd: 9

5. else Clause on Loops

This is unique to Python, and many newcomers find it surprising. A loop's else clause runs when the loop finishes normally — i.e., it was not terminated by break:

for n in range(2, 10):
    for x in range(2, n):
        if n % x == 0:
            print(f"{n} equals {x} * {n//x}")
            break
    else:
        # no factor found — n is prime
        print(f"{n} is a prime number")

# 2 is a prime number
# 3 is a prime number
# 4 equals 2 * 2
# 5 is a prime number
# 6 equals 2 * 3
# 7 is a prime number
# 8 equals 2 * 4
# 9 equals 3 * 3

The else belongs to the for (or while), not the if. Think of it as: "run this if no break happened."

Same pattern works with while:

n = 0
while n < 5:
    n += 1
    if n == 3:
        break
else:
    print("Completed without break")  # won't run — break at 3

6. enumerate() — Index + Value

enumerate() adds a counter to an iterable. It yields (index, value) tuples:

seasons = ["Spring", "Summer", "Fall", "Winter"]
for i, season in enumerate(seasons):
    print(f"{i}: {season}")

# 0: Spring
# 1: Summer
# 2: Fall
# 3: Winter

You can specify a starting value:

for i, season in enumerate(seasons, start=1):
    print(f"{i}: {season}")

# 1: Spring
# 2: Summer
# 3: Fall
# 4: Winter

This is always preferred over range(len(seq)) when you need both index and value.

7. zip() — Iterating Multiple Sequences

zip() pairs elements from two or more iterables, stopping at the shortest one:

names = ["Alice", "Bob", "Charlie"]
scores = [85, 92, 78]

for name, score in zip(names, scores):
    print(f"{name}: {score}")

# Alice: 85
# Bob: 92
# Charlie: 78

You can zip three or more:

questions = ["name", "quest", "color"]
answers = ["lancelot", "holy grail", "blue"]
for q, a in zip(questions, answers):
    print(f"What is your {q}? {a}")

With unequal lengths, zip() stops at the shortest. To keep all values, use itertools.zip_longest() (covered later).

8. Nested Loops

Loops inside loops are common for working with 2D data:

for row in range(3):
    for col in range(3):
        print(f"({row},{col})", end=" ")
    print()  # newline after each row

# (0,0) (0,1) (0,2)
# (1,0) (1,1) (1,2)
# (2,0) (2,1) (2,2)

Multiplication table:

for i in range(1, 4):
    for j in range(1, 4):
        print(f"{i}×{j}={i*j}", end="\t")
    print()

# 1×1=1   1×2=2   1×3=3
# 2×1=2   2×2=4   2×3=6
# 3×1=3   3×2=6   3×3=9

9. pass — Placeholder in Loops

pass does nothing. Useful as a placeholder when you need a loop body but haven't written the logic yet:

for item in items:
    pass  # TODO: process item later

while True:
    pass  # busy-wait for interrupt
